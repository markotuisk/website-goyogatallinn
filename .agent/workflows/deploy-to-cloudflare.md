# GoYoga Website — Cloudflare Pages Deployment Skill

## ⚠️ Key Architecture Understanding

This is a **static site generator architecture**:
- Source HTML files live in the root (e.g. `teacher.html`, `index.html`)
- `build_script.py` reads these source files + data from `js/*.js` and outputs translated static HTML to `en/`, `et/`, `fi/` subdirectories
- Cloudflare Pages is deployed **directly via Wrangler CLI** — the GitHub webhook is broken and should NOT be used
- All pre-built static HTML (`en/`, `et/`, `fi/`) **must be committed first**, then deployed via CLI

## Deployment Workflow

### Step 1 — Make content/data changes
- Edit teacher data in `js/teachers-detail.js`
- Edit translations in `js/data.js`
- Edit source HTML templates (e.g. `teacher.html`)

### Step 2 — Run the build script locally
```bash
# Activate the virtual environment first
source .venv/bin/activate

# Run the build script to regenerate static HTML in en/, et/, fi/
python3 build_script.py
```

The build script:
- Reads `js/data.js` (translations), `js/seo-data.js` (SEO/routes), `js/teachers-detail.js` (teacher data)
- Reads each `.html` file in the root
- For each language (`en`, `et`, `fi`): translates, applies URL routing, injects SEO schemas and GA4, and writes to the respective lang subdirectory
- Also regenerates `sitemap.xml`

### Step 3 — Commit ALL changed files (including built output)
```bash
git add .
git commit -m "your message here"
git push
```

> Critical: Make sure to also `git add` the `en/`, `et/`, `fi/` directories and `sitemap.xml`. These are the compiled outputs that Cloudflare Pages actually serves.


### Step 4 — Deploy via Wrangler CLI (NOT GitHub push — webhook is broken!)
```bash
# Load credentials
source .env

# Or set inline:
export NVM_DIR="$HOME/.nvm" && source "$NVM_DIR/nvm.sh"
CLOUDFLARE_API_TOKEN=yhn02GlF6iPiPiNoYQ8ZdARlGDB7oQD1ejcofZv0 \
CLOUDFLARE_ACCOUNT_ID=bc3f6c1145456bef0f781f62744c74ed \
wrangler pages deploy . --project-name goyoga-website --branch main
```

Expected output:
```
✨ Success! Uploaded N files (N already uploaded)
✨ Deployment complete! Take a peek over at https://XXXXX.goyoga-website.pages.dev
```

### Step 5 — Cache-bust JS data files if changed
If you changed any JS data file (e.g. `js/teachers-detail.js`), bump the `?v=N` version query on script tags in the source HTML (`teacher.html`) and rebuild:

```html
<!-- Bump ?v=2 to ?v=3 -->
<script src="js/teachers-detail.js?v=3"></script>
```

Then re-run Steps 2, 3, and 4.

## ⚠️ Troubleshooting: Cloudflare Not Picking Up New Commits

**Symptom**: Build log shows an old commit hash even after pushing new commits to `main`.

**Root cause**: Cloudflare Pages is configured with a wrong "Production Branch" in its dashboard settings, OR the GitHub ↔ Cloudflare webhook is broken/misconfigured.

**Fix (must be done manually in Cloudflare Dashboard)**:
1. Go to: `https://dash.cloudflare.com` → **Workers & Pages** → `goyoga-website`
2. Click **Settings** → **Builds & Deployments**
3. Verify **Production branch** is set to `main`
4. If the branch is correct but still deploying wrong commits: go to **Connected Git Repository** and check if the GitHub webhook is active
5. Re-trigger by clicking **Create New Deployment** → **Retry Deployment** from the latest commit

## File Reference

| Path | Purpose |
|------|---------|
| `js/teachers-detail.js` | Teacher profile data (bio, role, philosophy) — ALL 3 LANGUAGES |
| `js/data.js` | UI translations for all languages |
| `js/seo-data.js` | SEO metadata, URL route mappings per language |
| `teacher.html` | Source template for teacher profile page |
| `en/teacher.html` | Built English teacher page (auto-generated by build_script.py) |

| `fi/opettajat.html` | Built Finnish teacher page (auto-generated) |
| `build_script.py` | Build script — run locally, commit output |
| `requirements.txt` | Python deps for build_script.py |
| `wrangler.toml` | Cloudflare Pages project config |
| `_redirects` | Cloudflare language routing rules |
